# StS2 Blind Map (블라인드 맵)

맵의 모든 노드를 미지(`?`) 아이콘으로 가려, 액트 진행을 **포그 오브 워(fog-of-war) 챌린지**로 바꾸는 **Slay the Spire 2** 모드입니다.

> 경로를 *블라인드*로 선택합니다. 다음 노드가 상점일까, 엘리트일까, 쉬운 전투일까? 방에 들어가기 전까지는 알 수 없습니다.

## 이 모드는 무엇을 하나요

맵의 모든 노드를 게임 내장 미지(`?`) 스프라이트로 다시 그립니다. 단, 원래 경로 계획에 필요한 아래 노드만 **예외**로 그대로 표시합니다:

- **보스** — 항상 표시 (고정 목적지).
- **보스 직전 휴식(불)** — 보스전 앞에 보이는 그 모닥불.
- **시작 노드**.
- **고정 보물(유물) 노드** — 유물을 확정적으로 얻는 지점은 그대로 표시.

그 외 전부 — 몬스터, 엘리트, 상점, 이벤트, 다른 휴식 — 는 `?` 로 표시됩니다. 무엇이 있는지 모른 채 경로를 확정하고, 각 방은 도착해서야 정체가 드러납니다.

## 중요: 이건 포그 오브 워이지, 랜덤 방이 아닙니다

이 모드는 **방의 실제 정체를 바꾸지 않습니다.** 가려진 상점은 도착하면 여전히 상점이고, 엘리트는 여전히 엘리트입니다. *아이콘만* 가릴 뿐 — 방 종류, 인카운터, 보상, RNG 는 전혀 건드리지 않습니다. 맵 랜덤화가 아니라 순수 시각 오버레이입니다.

(노드를 진짜 "?" 이벤트 방으로 바꾸면 맵의 약 85% 가 랜덤 이벤트가 되고 액트의 엘리트/상점 구조가 사라지므로, 이 모드는 의도적으로 그렇게 하지 않습니다.)

## 동작 상세

- **지나치기 전까지 `?` 유지.** 진입하는 노드는 전투로 들어가는 페이드 동안, 그리고 방 안에 있는 동안 계속 가려져 있다가, *다음* 노드로 넘어가야 비로소 공개됩니다 — 한 박자 일찍 정체가 새지 않습니다.
- **지나온 길은 보입니다.** 이미 클리어하고 지나친 노드는 본모습을 드러내, 어디를 거쳐 왔는지 확인할 수 있습니다.
- **보스 / Ancient 노드는 게임의 별도 클래스가 그리므로** 손대지 않아 자동으로 표시됩니다.

## 작동 원리

1. `MegaCrit.Sts2.Core.Nodes.Screens.Map.NNormalMapPoint.UpdateIcon` 을 Harmony(HarmonyX, 게임 번들)로 패치합니다.
2. 게임이 노드의 실제 아이콘을 그린 뒤, postfix 가 노드의 `_icon` / `_outline` 텍스처를 `map_unknown` 스프라이트로 교체합니다 — 예외 노드(보스 직전 휴식, 고정 보물, 시작 노드)는 제외.
3. 노드의 `MapPointType` 은 **절대 변경하지 않으므로**, 방 결정·세이브·멀티플레이 동기화는 모두 변경 없는 맵을 봅니다.

## 멀티플레이

완전한 클라이언트 사이드 모드입니다. DLL 은 설치한 플레이어에게만 로드되고, 게임 상태에 쓰지 않으며, 네트워크 메시지를 보내지 않고, 방 내용을 바꾸지 않습니다 — 다른 플레이어는 변경 없는 게임을 보며 desync 가 없습니다. 매니페스트에 `"affects_gameplay": false` 로 표기되어 있습니다.

## 설치

1. [GitHub Releases](../../releases) (또는 Nexus 의 Files 탭) 에서 최신 `Sts2BlindMap-vX.Y.Z.zip` 을 다운로드.
2. `Sts2BlindMap/` 폴더를 다음 위치에 압축 해제:
   ```
   <Slay the Spire 2 설치 경로>/mods/
   ```
   최종적으로 다음 형태가 되어야 합니다:
   ```
   <Slay the Spire 2>/mods/Sts2BlindMap/Sts2BlindMap.dll
   <Slay the Spire 2>/mods/Sts2BlindMap/Sts2BlindMap.json
   ```
3. 게임 실행.

## 소스에서 빌드

요구사항:
- .NET SDK 9.0
- Godot.NET.Sdk 4.5.1 (자동 해결)
- 로컬 Slay the Spire 2 설치 (`Sts2PathDiscovery.props` 가 Steam 레지스트리/표준 경로로 자동 탐지)

```sh
dotnet build Sts2BlindMap.csproj -c Release
```

빌드 시 `Sts2BlindMap.dll` 과 `Sts2BlindMap.json` 이 `<sts2>/mods/Sts2BlindMap/` 로 자동 복사됩니다.

## 설정

게임 실행 전 환경 변수 `STS2_BLIND_MAP_DISABLED=1` 을 설정하면 모드를 제거하지 않고 블라인드 효과만 끌 수 있습니다.

## 주의사항 / 한계

- **보스 노드는 의도적으로 항상 표시** — 고정 목적지라 가릴 이유가 없습니다.
- **다른 맵 정보 모드는 노드 정체를 노출할 수 있습니다.** 노드 데이터를 직접 읽는 모드(예: 전투 노드에만 뜨는 포션 드롭 확률 배지)는 맵이 블라인드된 것을 모르므로, `?` 노드의 실제 정체를 누출할 수 있습니다. 의도한 챌린지를 위해서는 Blind Map 을 단독으로 쓰거나 다른 맵 오버레이 모드를 비활성화하세요.
- 시각 레이어에서 아이콘 텍스처만 교체합니다. 향후 게임 패치가 맵 포인트 클래스나 `map_unknown` 에셋 경로를 바꾸면 소스 갱신이 필요할 수 있습니다.
